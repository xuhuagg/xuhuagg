基于BDP数据可视化平台的知乎用户分析

置顶 | 发表于 2017-09-05 | 分类于 Java ， 爬虫 |
字数统计: 796

涉及

  • Java线程池、连接池
  • pc端、手机端抓包
  • Jsoup、正则分析
  • 数据库JDBC
  • BDP数据分析平台
  • 多知乎账号防封号

源码及数据库文件（配合代码看教程）：github

思路：获取知乎话题页ID，由话题ID得到分话题的ID，再由分话题ID得到关注了话题的用户的用户名，根据用户名模拟手机发包，得到用户信息（电脑端信息不完整），然后存入本地数据库,最后把数据导入BDP个人版，配置图表后可得到想要的统计图

程序流程

第一步：获取主话题ID

对应Main.java中的函数getTopicID()

  1. 进入https://www.zhihu.com/topics,右键话题审查元素

    获取完话题首页的话题ID后，程序进入下一步。

    第二步：获取分话题ID

    对应的类HandleTopic.java

  1. 如图所示，进入主话题页面如 https://www.zhihu.com/topics#互联网 ，每次下拉会刷新出20个分话题
  2. 分析post发送的包可以看出，topic_id对应话题ID，offset为偏移量，比如发送0，则返回0-20的分话题
  3. 每个主话题创建一个线程，发送包给服务器，并从返回的json信息中获得分话题ID,一共获得约1.6w个分话题ID

第三步：获得用户ID

对应GetUserUrl.java ,这一步需要设置post参数，登陆状态下才能访问下面的页面

  1. 要设置cookie 和X-Xsrftoken才能爬取，同样，分析post发送的包，每次下拉刷新20个用户
  2. 设置好后发包给服务器，从返回的response中提取出用户ID，因为程序运行13个小时的时候突然断网，只爬到了4.5w用户ID

第四步：模拟手机发包获取用户信息

对应GetUserInfo.java，为什么要模拟手机发包，因为pc端许多用户信息被隐藏了，而手机端可以访问

  1. 改ua头，推荐使用fiddler抓包（不懂手机抓包可百度），手机抓包状态访问知乎，得到对应的cookie和User-Agent
  2. 运行程序开始模拟手机发包，收到的response中得到需要的信息，最后一共获得大概4W用户数据，后面爬取速度越来越慢，我就直接终止程序了。

第五步：数据导入BDP数据分析平台

  1. 注册BDP个人版，可以直接导入MySQL表，但需要开会员，推荐把sql文件转为excel文件，再导入，这样免费。
  2. 进入BDP个人版网页，建立自己的表格，BDP有很多图表，选择适合的图表，设置坐标，就能得到你想要的统计图。

最后

  • 程序用到了jsoup、httpclient、JDBC需要自己添加jar包
  • 先前使用单账号，跑到几十分钟，爬不到数据，被知乎封了账号。后来问了人知道了多账号避免封号的方法，需要购买知乎账号，要使用EditThisCookie，登陆每个知乎账号，记录z_co,然后用EditThisCookie插件清除cookie记录，再登陆另外的账号，记录，知乎通过这个预判用户。
